# 查询示例
## 1. 子条件查询
根据特定字段查询所指特定值
### 1.1 Query Context
在查询过程中，除了判断文档是否满足查询条件以外，ES还会计算一个_score来标识匹配的程度，旨在判断目标文档和查询条件匹配的有多好。

## 2. 复合条件查询（Compound queries）
以一定的逻辑组合子条件查询，常用查询有固定分数查询、布尔查询等
1.固定分数查询（constant_score query）
URL: 149.28.229.144:9200/_search  
Method: POST  
Content-Type: json/application  
content:
```json
{
	"query": {
		"constant_score": {
			"filter": {
				"match": {
					"title": "ElasticSearch"
				}
			},
			"boost": 2
		}
	}
}
```

2.布尔查询（bool query）
关键词有：must、filter(和must效果一样，不过不计算分值，会有缓存)、should（类似于或）、must_not
URL: 149.28.229.144:9200/_search  
Method: POST  
Content-Type: json/application  
content:
```json
{
	"query": {
		"bool": {
			"should": [
				{
					"match": {
						"author": "ashin"
					}
				},
				{
					"match": {
						"title": "ElasticSearch"
					}
				}
			],
			"filter": [
				{
					"term": {
						"word_count": 9000
					}
				}
			]
		}
	}
}
```
返回json
```json
{
    "took": 9,
    "timed_out": false,
    "_shards": {

        "successful": 8,
        "skipped": 0,
        "failed": 0

        "total": 1,

        "hits": [
            {
                "_index": "book",
                "_type": "novel",
```
